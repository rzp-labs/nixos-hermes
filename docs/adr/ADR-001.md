# ADR-001: Immutable User Configuration

**Status:** Accepted

## Context

NixOS supports both mutable (imperative) and immutable (declarative) user management. Mutable users allow `useradd` and `passwd` on the host but create drift between what Nix declares and what is on disk.

## Decision

`users.mutableUsers = false`. All user accounts and authorized SSH keys for interactive login are declared in `modules/users.nix`. No password-based login; authentication is SSH key only.

## Consequences

No users can be added imperatively on the host. Adding a user requires a NixOS rebuild. This eliminates an entire class of configuration drift.
